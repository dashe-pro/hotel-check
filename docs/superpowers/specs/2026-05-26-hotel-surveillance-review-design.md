# 酒店偷拍信息聚合小程序 · 设计方案

日期：2026-05-26

---

## 一、产品定位

用户在订酒店前，搜一下看有没有人在这家酒店遇到过偷拍。

- 不做酒店综合点评，不做摄像头检测工具
- 只做一件事：酒店偷拍信息聚合
- 信息分两类：公开案件记录 + 用户提交评论
- 平台不下结论，展示原始数据，用户自己判断

---

## 二、用户流程

```
首页（搜索框 + 最近动态）
    → 搜索结果列表（模糊匹配酒店名）
    → 酒店详情页
        ├── 公开案件记录（如有，红色标签标注来源）
        └── 用户评论列表（时间倒序）
    → 提交评论（需微信授权登录，前置审核）
```

共3个页面 + 1个提交页 + 1个"我的"页面。

---

## 三、技术栈

- **前端**：uni-app + Vue，编译发布到微信小程序
- **后端**：微信云开发（云函数 + 云数据库 + 云存储）
- **酒店数据**：高德地图 POI API（5000次/月免费额度）
- **内容安全**：微信 msgSecCheck（文本）+ imgSecCheck（图片）

---

## 四、数据模型

### hotels 集合

| 字段 | 类型 | 说明 |
|------|------|------|
| `_id` | string | 自动生成 |
| `name` | string | 酒店名称 |
| `address` | string | 城市+区 |
| `hasCase` | boolean | 是否有公开案件记录 |
| `caseCount` | number | 案件数 |
| `reviewCount` | number | 评论数 |
| `createdAt` | date | 创建时间 |

### reviews 集合

| 字段 | 类型 | 说明 |
|------|------|------|
| `_id` | string | 自动生成 |
| `hotelId` | string | 关联酒店ID |
| `userId` | string | 微信openid |
| `type` | string | "case" / "user" |
| `content` | string | 文字内容 |
| `discoveryDate` | date | 发现日期 |
| `images` | string[] | 截图，可选，最多3张 |
| `source` | string | 案件来源链接（仅 case） |
| `status` | string | "pending" / "approved" / "rejected" |
| `createdAt` | date | 提交时间 |

---

## 五、内容安全（前置审核模式）

```
用户提交 → 前端敏感词过滤 → 微信 API 审核
    → 全部进入待审队列（不直接发布）
    → 人工确认后发布
```

- 提交页和详情页始终展示免责声明
- 人工审核通过云开发控制台操作，不搭建后台系统

---

## 六、冷启动策略

1. **高德 API 自动录入**：按城市批量搜索"酒店"POI，每周增量同步
2. **公开案件聚合**：云函数定时抓取新闻关键词，人工确认后入库
3. **用户搜不到时**：实时调高德 API 补录酒店

---

## 七、MVP 功能清单

**前端页面：**
- 首页（搜索框 + 最新评论动态）
- 搜索列表（模糊匹配）
- 酒店详情（案件 + 评论）
- 提交评论（含免责声明）
- 我的（登录 + 我的评论）

**后端云函数：**
- search-hotels
- get-hotel-detail
- submit-review
- sync-hotels（定时任务）

**运营后台：**
- 云开发控制台直接操作数据库审核评论和录入案件

---

## 八、变现路径

| 阶段 | 方式 |
|------|------|
| 冷启动 | 不变现，专注内容积累 |
| 增长期 | 流量主广告 + OTA 分销佣金 |
| 规模化 | 保险合作 + 自有电商 |

初期使用个人主体注册，需要变现时升级为企业主体。

---

## 九、法律风险与应对

| 风险 | 应对 |
|------|------|
| 酒店发律师函要求删帖 | "通知-删除"机制，24小时内处理 |
| 用户虚假举报 | 前置审核 + 免责声明 + 证据截图 |
| 微信内容安全 | msgSecCheck + 敏感词过滤 |
| 传播他人隐私 | 上传图片自动脱敏 |

平台定位为信息聚合，不下结论不做判定，免责声明始终可见。
